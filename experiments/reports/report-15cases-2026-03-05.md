## 핵심 발견

### 1. 난이도-효과 상관관계 (최대 발견)

```
Delta = f(난이도)

기본 (5 cases)      +23.8  ████████████████████████
고급 (5 cases)      +29.6  ██████████████████████████████
초고난이도 (5 cases) +36.2  ████████████████████████████████████
```

**프로젝트가 복잡할수록 Harness 효과가 극대화된다.**
- 기본 과제: Claude가 이미 잘 하는 영역 → Delta 적당
- 복합 시스템: 아키텍처 설계가 품질 전체를 좌우 → Delta 극대

### 2. 테스트 커버리지가 최대 격차 차원 (+4.9)

Baseline 평균 2.5점은 "테스트 거의 없음" 수준. Harness의 agents에 테스트 전담 에이전트가 포함되어 체계적 테스트 생성.

### 3. 아키텍처가 두 번째 격차 (+4.4)

Baseline은 단일/2파일 모놀리식 경향 (평균 3.9점). CLAUDE.md의 파일 구조 사전 정의가 모듈 분리를 강제.

### 4. 기능 완성도는 격차가 상대적으로 적음 (+2.8)

Claude의 기본 구현 능력은 충분. 차이를 만드는 것은 "무엇을 구현하느냐"가 아니라 "어떻게 구현하느냐".

### 5. Skills의 참조 문서가 정확성을 결정

초고난이도 케이스에서 Baseline correctness 평균 4.8점 vs Harness 8.2점. Raft RPC 명세, Opcode 참조, LSP 프로토콜 등의 references/가 알고리즘 정확성을 보장.

### 6. 최대 Delta 케이스: Case 015 LSP (+39)

LSP는 프로토콜 스펙, 증분 파싱, 에러 복구, 심볼 분석 등 다층적 복잡성. Baseline에서 가장 낮은 40점. Harness의 5계층 아키텍처 + 상세 프로토콜 참조가 결정적 차이.

### 7. 최소 Delta 케이스: Case 004 README (+16)

문서 작성은 Claude의 기본 능력으로 충분. 구조화 이점이 적은 영역.

## 결론

> **Harness(.claude/ 사전 구성)는 Claude Code 산출물을 49.5점에서 79.3점으로 끌어올리며(+60%),
> 테스트 커버리지(+4.9), 아키텍처(+4.4), 에러 처리(+3.0) 차원에서 결정적 차이를 만든다.
> 초고난이도 복합 시스템에서 효과가 극대화(+36.2)되며, 이는 Skills의 알고리즘 참조와
> Agents의 역할 분리가 구조화된 고품질 산출물을 보장하기 때문이다.**
