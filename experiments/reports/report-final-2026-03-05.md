### 고급 패턴의 Skills -> 코드 직접 전이

고난이도 케이스에서 Skills에 정의된 알고리즘/패턴이 코드에 직접 반영된 사례:

- **Case-007**: Pratt Parser 패턴 -> 연산자 우선순위 우아한 처리, AST 팩토리 함수
- **Case-008**: SAGA Orchestrator steps/compensation 패턴 -> 범용 재사용 가능 설계, 커스텀 에러 계층

- **Case-010**: RGA 알고리즘 + 인과성 버퍼 -> 이중 연결 리스트 + nodeMap, pendingInserts, VectorClock 모듈

### Baseline의 고난이도 한계: "단일 파일 모놀리식" 경향 심화

### 핵심 메시지

> **Harness(.claude/ 사전 구성)는 "복잡한 프로젝트일수록 더 가치 있다."**

- 기본 태스크에서도 일관되게 우위 (+7.0)
- 고난이도 태스크에서 효과 증폭 (+8.25)
- 구조화와 코드 품질에서 가장 큰 영향
- Skills를 통한 고급 패턴/알고리즘 전이가 고난이도에서 결정적 차이

### 권장사항

1. **모든 프로젝트에 .claude/ 구성 권장**: 기본 태스크에서도 +7.0의 일관된 효과
2. **복잡한 프로젝트에서 필수**: 고난이도에서 +8.25 -- 아키텍처 가이드 없이는 모놀리식 경향 심화
3. **Skills에 도메인 특화 패턴 정의**: Pratt Parser, SAGA, CRDT 등 전문 알고리즘이 코드에 직접 전이됨
4. **Commands로 작업 프로세스 표준화**: "분석 -> 설계 -> 구현 -> 테스트"의 체계적 절차가 품질 보장
5. **구조화 가이드를 최우선**: 가장 큰 Delta(+3.30)를 생성하는 차원이므로 CLAUDE.md에 아키텍처/모듈 분리 규칙을 반드시 포함
