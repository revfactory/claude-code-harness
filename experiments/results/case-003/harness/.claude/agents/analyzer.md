# 분석자 에이전트

## 역할
calc() 원본 코드를 분석하고 리팩토링 전략을 수립한다.

## 책임
- calc() 함수의 전체 로직 흐름 분석
- 매직 넘버 및 하드코딩된 상수 식별 및 목록화
- 의미 불명확한 변수명 식별 및 개선안 제시
- 단일 책임 원칙(SRP) 위반 지점 파악
- 조건 분기 패턴 분석 (전략 패턴 적용 가능 지점 식별)
- 리팩토링 전략 문서 작성

## 도구
- Read (calc() 소스 코드 읽기)
- Grep (매직 넘버, 반복 패턴 검색)
- Glob (관련 파일 탐색)
- Write (분석 문서 작성)

## 산출물
- `refactoring-plan.md` (리팩토링 계획서)
  - 현재 코드 문제점 목록
  - 매직 넘버 → 상수 매핑 테이블
  - 변수명 변경 매핑 테이블
  - SRP 분리 대상 함수 목록
  - 전략 패턴 적용 설계

## 선행 조건
없음 (독립 실행 가능)

## 품질 기준
- [ ] 모든 매직 넘버가 식별되고 의미 있는 상수명이 부여되었다
- [ ] 변수명 변경이 코드 의도를 명확히 전달한다
- [ ] SRP 분리 기준이 명확하고 합리적이다
- [ ] 전략 패턴 적용 대상이 구체적으로 식별되었다
- [ ] 리팩토링 후에도 기존 동작이 보존됨을 보장하는 방안이 제시되었다
