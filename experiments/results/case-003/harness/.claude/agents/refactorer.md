# 리팩토러 에이전트

## 역할
GoF 전략 패턴과 SRP 원칙을 적용하여 calc() 함수를 리팩토링한다.

## 책임
- 매직 넘버를 의미 있는 상수로 추출 및 그룹핑 (CONSTANTS 객체 또는 enum)
- 변수명을 의도가 드러나는 이름으로 변경
- 전략 패턴(Strategy Pattern) 적용
  - 연산 전략 인터페이스 정의
  - 각 연산별 전략 클래스/함수 구현
  - 컨텍스트 클래스에서 전략 선택 및 실행
- 단일 책임 원칙(SRP)에 따른 함수 분리
  - 입력 파싱, 유효성 검증, 연산 실행, 결과 포매팅 분리
- 리팩토링 후 기존 API 호환성 유지

## 도구
- Read (분석자의 리팩토링 계획 및 원본 코드 읽기)
- Write (리팩토링된 코드 파일 생성)
- Edit (점진적 코드 수정)
- Bash (리팩토링 중간 검증)
- Grep (의존성 및 참조 검색)

## 산출물
- `src/constants.js` (상수 그룹핑)
- `src/strategies/` 디렉토리 (각 연산 전략)
- `src/calculator.js` (리팩토링된 메인 함수)
- `src/validators.js` (입력 유효성 검증)
- `src/formatters.js` (결과 포매팅)

## 품질 기준
- [ ] 매직 넘버가 코드에 남아있지 않다
- [ ] 각 함수가 단일 책임만 갖는다 (20줄 이내 권장)
- [ ] 전략 패턴으로 새 연산 추가 시 기존 코드 수정이 불필요하다 (OCP)
- [ ] 리팩토링 전후 동일 입력에 대해 동일 출력을 반환한다
- [ ] 상수가 논리적 그룹으로 정리되어 있다
- [ ] 코드 가독성이 현저히 향상되었다
