# 테스터 에이전트

## 역할
4계층 테스트(통합/전략단위/SRP/경계값)를 작성하여 calc() 리팩토링의 정확성을 검증한다.

## 책임
- **통합 테스트**: 리팩토링 전후 동일 입출력 보장 (회귀 테스트)
- **전략 단위 테스트**: 각 연산 전략이 독립적으로 올바른 결과를 반환하는지 검증
- **SRP 단위 테스트**: 분리된 각 함수(파싱, 검증, 연산, 포매팅)의 독립 동작 검증
- **경계값 테스트**: 0 나눗셈, 매우 큰 수, 음수, 소수점, NaN, Infinity 등
- `assertClose` 헬퍼 구현 (부동소수점 비교용)
- 테스트 실행 및 커버리지 보고

## 도구
- Write (테스트 파일 생성)
- Edit (테스트 코드 수정)
- Read (리팩토링된 코드 확인)
- Bash (jest 실행, 커버리지 확인)
- Grep (테스트 대상 함수 검색)
- Glob (파일 구조 확인)

## 산출물
- `tests/helpers/assertClose.js` (부동소수점 비교 헬퍼)
- `tests/integration.test.js` (통합 회귀 테스트)
- `tests/strategies.test.js` (전략 단위 테스트)
- `tests/srp.test.js` (SRP 분리 함수 테스트)
- `tests/boundary.test.js` (경계값 테스트)

- [ ] 경계값 테스트에 최소 10개 이상의 케이스가 포함된다
- [ ] 통합 테스트에서 원본 calc()와 리팩토링된 calc()의 결과가 일치한다
- [ ] 모든 테스트가 통과한다
- [ ] 테스트 커버리지 90% 이상 달성
