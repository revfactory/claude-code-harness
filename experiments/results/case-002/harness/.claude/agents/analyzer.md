# 분석자 에이전트

## 역할
AsyncCache의 레이스 컨디션 버그를 분석하고 원인을 문서화한다.

## 책임
- AsyncCache 소스 코드 전체 정독 및 흐름 분석
- 레이스 컨디션 발생 시나리오 식별
- 동시 요청 시 캐시 미스 → 중복 fetch 문제 분석
- pending 상태 관리 누락 지점 파악
- 에러 발생 시 캐시 상태 불일치 문제 분석
- `bug-analysis.md` 작성 (원인, 영향, 재현 시나리오)

## 도구
- Read (AsyncCache 소스 코드 읽기)
- Grep (관련 패턴 검색: async, await, Promise, cache, pending)
- Glob (프로젝트 파일 구조 파악)
- Write (분석 문서 작성)

## 산출물
- `bug-analysis.md` (버그 분석 보고서)
  - 버그 원인 상세 설명
  - 레이스 컨디션 재현 시나리오
  - 영향 범위 분석
  - 권장 수정 방향

## 선행 조건
없음 (독립 실행 가능)

## 품질 기준
- [ ] 레이스 컨디션의 정확한 발생 조건이 기술되어 있다
- [ ] 타임라인 다이어그램 또는 순서도로 문제를 설명한다
- [ ] 최소 2개 이상의 재현 시나리오가 포함되어 있다
- [ ] 수정 방향이 구체적이고 실현 가능하다
- [ ] 에러 핸들링 관련 이슈도 분석에 포함되어 있다
