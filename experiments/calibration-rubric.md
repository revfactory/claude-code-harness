# 평가 보정 기준 (Calibration Rubric)

## 문제 진단

### 편향 1: Baseline 완성도/정확성 과대평가
- 테스트 미포함 산출물이 completeness 8-9점 → 테스트는 완성도의 핵심 구성요소
- 알려진 버그(SQL NULL, 서비스 격리 위반)가 correctness 8점 → 과대

### 편향 2: Harness 사소한 결함에 과도한 감점
- Boolean() 강제변환 같은 경미한 이슈로 accuracy 7점 (-3)
- Case-006에서 code_quality가 Baseline보다 낮음 (7 vs 9) — 구조적 접근 불이익

### 편향 3: Structure 기준 불균일
- Baseline 단일파일이 4-6점으로 범위가 넓음 → 일관된 기준 필요
- Harness 체계적 구조가 9-10점으로 적절하나, Baseline 대비 상대적 격차 축소

### 편향 4: Case-004 점수 체계 불일치
- 가중 평균(7.0/8.2)을 5차원 합산(36/42)으로 변환한 근거 불명

## 보정된 채점 기준 (Anchored Rubric)

### Completeness (완성도)
- 10: 모든 요구사항 + 테스트 + 문서 + 추가 기능
- 9: 모든 요구사항 + 포괄적 테스트
- 8: 모든 요구사항 + 기본 테스트
- 7: 핵심 요구사항 충족 + 테스트 부재
- 6: 핵심 요구사항 충족, 일부 누락
- 5이하: 주요 요구사항 미충족

### Code Quality (코드 품질)
- 10: SOLID 완벽 준수, 디자인 패턴, 에러 계층, 현대적 문법
- 9: SOLID 대부분 준수, 적절한 패턴 적용, DRY
- 8: 깔끔한 코드, 일부 패턴 적용
- 7: 가독성 양호하나 중복/결합도 존재
- 6: 기능은 동작하나 코드 스멜 다수
- 5이하: 스파게티 코드, 심각한 중복

### Efficiency (효율성)
- 10: 최적 알고리즘/자료구조, 최소 의존성, 최적 코드량
- 9: 적절한 최적화, 효율적 구조
- 8: 불필요한 복잡도 없음, 적절한 코드량
- 7: 약간의 비효율 존재
- 6: 눈에 띄는 비효율 (O(n) vs O(1) 가능)
- 5이하: 심각한 성능 문제

### Correctness (정확성)
- 10: 모든 테스트 통과 + 엣지케이스 + 퍼즈 테스트 + 표준 완벽 준수
- 9: 모든 테스트 통과 + 엣지케이스 처리 + 표준 준수
- 8: 모든 테스트 통과 + 주요 엣지케이스
- 7: 기본 테스트 통과, 일부 엣지케이스 미처리
- 6: 알려진 버그 존재 (표준 위반 등)
- 5이하: 핵심 기능에 버그

### Structure (구조화)
- 10: 교과서적 아키텍처, 완벽한 관심사 분리, 테스트 계층화
- 9: 명확한 모듈 분리, 적절한 디렉토리 구조, 테스트 분리
- 8: 적절한 파일 분리, 기본 구조 존재
- 7: 다중 파일이나 구조가 불완전
- 6: 소수 파일, 플랫 구조
- 5: 2-3파일, 분리 최소
- 4이하: 단일 파일 모놀리식

## 핵심 보정 원칙
1. **테스트 미포함 = completeness 최대 7점** (테스트는 필수 산출물)
2. **알려진 표준 위반 버그 = correctness 최대 6점** (SQL NULL 등)
3. **사소한 결함(Boolean 강제변환 등) = 최대 1점 감점** (accuracy 9→8)
4. **단일 파일 = structure 최대 4점** (관심사 분리 전무)
5. **서비스 격리 위반 = code_quality 최대 6점** (아키텍처 원칙 위반)
