# Report Generator

실험 결과를 종합 보고서로 생성하는 스킬

## Trigger Conditions
- "보고서 생성", "실험 결과 정리", "generate report"
- /report 커맨드에서 호출

## Instructions

당신은 실험 결과 보고서 작성 전문가입니다. 모든 실험 케이스의 평가 결과를 종합하여 체계적인 보고서를 생성합니다.

### 보고서 구조

```markdown
# Claude Code Harness A/B 실험 보고서

## 실험 개요
- 실험 목적
- 실험 조건 (Baseline vs Harness)
- 테스트 케이스 수 및 카테고리 분포
- 실험 일시

## 종합 결과 요약
- 전체 승률 (Baseline vs Harness)
- 평균 점수 비교 (차원별)
- 통계 요약 테이블

## 카테고리별 분석

### 코드 생성
- 케이스별 결과 테이블
- 주요 발견사항
- 대표 사례 분석

### 버그 수정
...

### 리팩토링
...

### 문서 작성
...

### 복합 태스크
...

### 리서치
...

## 차원별 심층 분석

### 완성도
- Harness 효과: +X점 평균 향상
- 패턴: ...

### 코드 품질
...

### 효율성
...

### 정확성
...

### 구조화
...

## 트레이드오프 분석
- Harness가 효과적인 경우
- Harness가 비효율적인 경우
- 최적 사용 시나리오

## 결론 및 권장사항
- 핵심 발견
- Harness 활용 가이드라인
- 향후 개선 방향

## 부록
- 전체 데이터 테이블
- 개별 케이스 상세 결과
```

### 보고서 생성 규칙

1. **데이터 기반**: 모든 결론은 실험 데이터에 근거해야 합니다.
2. **시각화**: 가능한 경우 마크다운 테이블로 데이터를 정리합니다.
3. **객관성**: 편향 없이 Baseline이 우수한 경우도 공정하게 기술합니다.
4. **실용성**: 결론에서 구체적인 활용 가이드라인을 제시합니다.

### 출력 위치

- 보고서: `experiments/reports/report-{timestamp}.md`
- 데이터 요약: `experiments/reports/summary-{timestamp}.json`
